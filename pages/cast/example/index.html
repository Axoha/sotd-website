Directory structure

It is my humble opinion, that for every character entry you should make a new folder inside of the cast folder (like I made this example folder), and inside it you create an index.html file (this one) and an assets folder, inside this assets folder you'll place all the pictures from the carousel, if you follow this structure you won't need to change a thing about the paths to the img tags in this template, just the name of the file itself (so where it says, for example <img> src="./assets/FerdRayMonochromeEra.png"> </img> you can just replace the FerdRayMonochromeEra.png by whatever the file you uploaded is called)

Here are some tags you might want to use

I don't know what your html level is, you might not need these, if you need more help than this I recommend w3schools html and css guides or just hit me up

h1 through h6

These make like... titles, they're big, h1 is the biggest h6 is the smallest, it's typically suggested that you don't use more than one h1 in a page, and we already use one on characterName, so maybe only use h2 onwards if you care about that stuff

p

Stands for paragraph, they're the bread and butter, it'll start in a new line, put anything between p and /p and it will show up as text on the screen with the regular font-size

img

This is an image, you need to give this an src attribute, it will look like this:
<img src="./assets/NameOfWhateverImageYouDroppedIntoTheAssetsFolder.png"> it's also worth noting that unlike the previous tags, this one doesn't come accompanied by a closing tag

ul and li

ul mean unordered list, that's how you make the little bullet points, inside the ul (so between ul and /ul) you put various li (list item), whatever is contained in one li will make up one bullet point (this sounds confusing just follow the example list I put above)

br

You can put a br in the middle of text just to cause a line break, like here:

This text goes on and on and on and
until it doesn't

This tag also doesn't have a closing tag, so you'll use <br> without any </br>
